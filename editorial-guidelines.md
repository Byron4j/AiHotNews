## 1. 选题原则

一篇选题能否立项，要同时通过 **「3T 检验」**：

1. **Timeliness（时效）**：产品/事件在 7 天内有显著新动作（发布、融资、改版、出圈）。
2. **Tension（张力）**：存在值得讨论的"为什么是它""谁会被颠覆""壁垒在哪"。
3. **Takeaway（可带走价值）**：读者读完能获得 ≥1 条可以执行的判断或动作。

只满足"火"但缺乏张力或可带走价值的，归入「快讯卡片」，不占主篇位置。

- slug：3–6 个英文单词，连字符分隔，描述文章核心论点而非产品本身

## 4. 写作风格规范

### 4.1 句式

- **短句优先**：一句话表达一个意思，平均不超过 25 字
- **一段不超过 4 行**：避免成块文字劝退读者
- **多用设问、对比、类比**：增加节奏感
- **避免"翻译腔"**：不写"作为一款……的产品""值得我们注意的是……"

### 4.2 术语

- 关键术语首次出现采用 **「中文（English）」** 格式
  - 例：编排器（Orchestrator）、人在回路（Human-in-the-loop, HITL）、智能体卡片（Agent Card）
- 缩写首次出现需展开：MCP（Model Context Protocol）

### 4.3 禁用词与慎用词

| 禁用 | 替换思路 |
| --- | --- |
| AI 万能 / 颠覆一切 / 革命性 | 用具体场景说话 |
| 史上最强 / 王炸 / 炸裂 | 标题党词汇，伤害专业度 |
| "我觉得"频繁出现 | 改为"判断是""结论是" |
| 通篇 emoji | 仅在「博主点评」区可用 ✅ ⚠️ 🎯 三个符号 |

### 4.4 数据与陈述

- 任何数字（融资额、用户数、性能指标）必须**带来源链接**
- 不能交叉验证的，明确标注：「据官方介绍」「未获独立验证」
- 引用官方原话用 `>` 引用块 + 链接

## 5. 事实核查（Fact-check）红线

发文前**必须完成**以下核查，缺一不可：

- [ ] **三源验证**：核心事实有 ≥2 个独立来源
- [ ] **官方核对**：产品功能描述与官网最新版本一致
- [ ] **时间戳**：避免使用"最近""不久前"，统一改为绝对日期
- [ ] **链接有效**：所有外链可访问
- [ ] **同名消歧**：产品名重名时（如本目录首篇 Mindra 就有 4 家同名），文中明确标识"我们讨论的是 mindra.co"

---

## 6. 引用规范

- 文末「参考来源」用 markdown 链接列表，按"官方 → 第三方分析 → 数据/榜单"排序
- 优先引用：
  - 产品官网、官方博客、官方文档
  - 一手融资公告（PR Newswire、官方推文）
  - 知名第三方（TechCrunch、The Information、机器之心、量子位）
  - 一手开源仓库（GitHub）
- **避免引用**：营销稿、AI 生成的聚合稿、未注明来源的二手翻译

## 8. SEO 与可发现性

- 标题：≤30 字，必须含**产品名**和**核心冲突点**
  - ✅ 好：`Mindra 拿下 120 万美元：这次轮到 Agent 团队"上岗打卡"`
  - ❌ 差：`一款很有意思的 AI 产品分享`
- 摘要（TL;DR）：≤150 字，能独立成立（脱离全文也能读懂）
- Tags：3–6 个，混合「赛道词」「技术词」「场景词」
  - 例：`agentic`, `orchestration`, `marketing-automation`, `A2A`

## 10. 更新机制

- 当文章涉及的事实发生重大变化（融资、改名、停服、争议事件），**不删旧文**，在文末 `Update Log` 追加日期与变化。
- 如修正错误，使用 `~~删除线~~` 保留原句，旁注修正版本。
- 季度回顾：每季度末抽查 5 篇，检查链接失效、过时判断，统一在 Update Log 中更新。

## 11. AI 协作规约（与 Claude Code 协作时）

为保证多人/人机协作的一致性：

- **写作前**：把本规范文件 `editorial-guidelines.md` 整体丢给 AI 作为上下文
- **过程中**：要求 AI 严格遵循八大模块，不允许"自由发挥"
- **生成后**：人工逐项过 §9 检查清单，**不放过任何一条事实未验证项**
- **判断与立场**：AI 可以协助起草「博主点评」的多种角度，但**最终立场由作者拍板**，AI 不得替作者表达情绪化结论

## 12. 日报研究规范（Daily Research Workflow）

### 12.1 信息源矩阵

日报编辑**必须**从以下多平台交叉获取信息，不得仅依赖单一来源：

**海外主流平台（优先获取一手信息）**

| 平台 | 用途 | 访问方式 |
|------|------|----------|
| x.com (Twitter) | 官方公告、研究员动态、AI实验室突发新闻 | 本地 ClashX Pro 代理 + curl |
| Reddit | r/MachineLearning / r/singularity / r/LocalLLaMA 社区反应和一手发现 | 本地 ClashX Pro 代理 + curl |
| YouTube | 主题演讲、产品演示、技术深度解读 | 本地 ClashX Pro 代理 + curl |
| The Verge | AI 产业深度报道、政策/法律跟踪 | 本地代理 + WebFetch/curl |
| TechCrunch | 融资、创业公司、产品发布 | 本地代理 + WebFetch/curl |
| CNBC | AI与资本市场交叉报道 | 本地代理 + WebFetch/curl |
| The Information | 独家深度、企业内幕 | 本地代理 + WebFetch/curl |

**国内平台**

| 平台 | 用途 |
|------|------|
| 量子位（qbitai.com） | 国内 AI 产业动态首发，海外新闻中文解读 |
| 机器之心（jiqizhixin.com） | AI 技术深度研究、行业通讯 |
| 36氪（36kr.com） | 创业公司融资、商业落地报道 |

**官方一手来源（优先于所有聚合渠道）**

- Anthropic 官方博客（anthropic.com/blog）
- OpenAI 官方博客（openai.com/index）
- Google DeepMind 博客
- DeepSeek、Kimi (Moonshot)、百度、阿里千问官方发布渠道
- GitHub 开源仓库（GitHub Trending、Release Notes）
- arxiv 论文预印本

### 12.2 时间窗口

- **前一天尾盘**：汇总前一天下午至夜间海外发布的信息（海外平台常在北京时间夜间发布）
- **当日早盘**：汇总当日清晨至上午的中文和海外信息
- 两段时间窗口的信息共同构成当日日报内容

### 12.3 重点追踪实体

日报编辑应优先寻找以下实体的第一手海外信源，而非仅依赖中文聚合报道：

- **Anthropic** — Claude 模型更新、安全/可解释性研究、API 变更、Claude Code 发布
- **OpenAI** — 模型发布（GPT-x）、API 更新、语音/Agent 产品、治理/法律动态
- **Musk / xAI / SpaceXAI** — Musk-Altman 诉讼、SpaceXAI 动态、Colossus 数据中心、Terafab 芯片厂
- **DeepSeek、Kimi (Moonshot)、百度、阿里千问** — 中国核心 AI 玩家，发布节奏快

### 12.4 市场局势分析（必含板块）

每期日报**必须**包含「市场局势分析」板块，覆盖以下四个维度：

- **一级市场**：本周/本日融资事件汇总（估值、金额、赛道），资本逻辑变化判断
- **二级市场**：AI相关股票的重大波动、IPO事件、财报信号，资金轮动方向
- **劳动力市场**：AI替代就业的标志性事件（上市公司裁员/缩编/效率提升等）
- **地缘资本**：中美AI资本流向、芯片供应链变化、跨境投资/监管动态

数据来源：CNBC、TechCrunch、The Information（海外）；qbitai、36kr、机器之心（国内）。

### 12.5 海外平台技术访问说明

海外开放平台（x.com、Reddit、YouTube、The Verge 等）为技术研究目的，通过本地 ClashX Pro 代理访问。访问工具链：

```
curl -sL --max-time <seconds> "<URL>" | <extract>
```

- 对 JavaScript 渲染型站点（The Verge、TechCrunch 等），通过 curl 获取原始 HTML + Python/grep 提取关键字段
- 对 API 型平台（Reddit），优先尝试 JSON 端点
- `.claude/settings.local.json` 中需提前注册所需域名
